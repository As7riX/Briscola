Visual C++ per lo sviluppo di applicazioni Linux

Configurazione del progetto per lo sviluppo con Linux

Questo carico di lavoro consente di creare codice C++ per server, desktop e dispositivi Linux. È possibile gestire le connessioni a questi computer da Visual Studio. Visual Studio copia automaticamente e compila in remoto i file del codice sorgente e può avviare l'applicazione con il debugger. Il sistema di progetto supporta l'uso di architetture specifiche come destinazione, tra cui ARM.

Connessione a Linux

Prerequisiti

Al momento la compilazione remota è supportata solo nel computer di destinazione Linux. La limitazione non dipende da distribuzioni specifiche di Linux, ma dalla presenza di alcuni strumenti dipendenti. In particolare, sono necessari openssh-server, g++, gdb e gdbserver. Usare l'utilità di gestione pacchetti preferita per installarli. Ad esempio, in sistemi basati su Debian, usare sudo apt-get install openssh-server g++ gdb gdbserver

Prima connessione

La prima volta che si imposta un computer Linux come destinazione, verranno richieste le informazioni sulla connessione. Viene attivato dalla compilazione del progetto.

Aggiunta e rimozione di connessioni

Per aggiungere una nuova connessione, passare a Strumenti > Opzioni e cercare Connessione. Gestione connessioni si trova in Multipiattaforma. Da tale posizione è possibile aggiungere e rimuovere connessioni.

Per cambiare la connessione usata da un progetto, passare alle proprietà del progetto e nelle impostazioni generali aggiornare l'opzione Computer di compilazione remota.

Proprietà del progetto

Tutte le opzioni necessarie per controllare la compilazione C++ sono esposte nella pagina delle proprietà del progetto. Verranno trattati solo alcuni aspetti della configurazione per Linux. In primo luogo, si noterà che nelle impostazioni generali la radice remota è impostata su ~/projects/ per impostazione predefinita, ma la directory di progetto remota verrà modificata in modo che corrisponda al nome del progetto in tale percorso.

Osservando le impostazioni generali del progetto è possibile notare la configurazione delle directory di output e intermedia. Si noterà anche che questo progetto è stato configurato come applicazione, di conseguenza l'eseguibile è ConsoleApplication1.out e si trova in bin/x64/Debug/. Tenere presente che come tipi di configurazione sono supportate anche le librerie statiche e dinamiche.

Aggiungere dipendenze aggiuntive alle librerie nella pagina delle proprietà Linker > Input.

È possibile passare al debugger comandi di pre-avvio aggiuntivi per eseguire operazioni quali l'avvio di app grafiche nel computer Linux remoto.

È anche possibile inviare eventi di post-compilazione per controllare il comportamento remoto, come in questo esempio che esporta un PIN GPIO per l'uso senza richiedere l'esecuzione dell'eseguibile come utente con privilegi avanzati.
